# 🎤 Smart Assistant - 발표용 요약 자료

## 📋 프로젝트 개요 (30초)

**Smart Assistant**는 PM을 위한 AI 기반 업무 자동화 도구입니다.

- **문제**: PM은 하루에 수백 개의 이메일/메시지를 받지만, 중요한 TODO를 놓치기 쉽습니다
- **해결**: LLM 기반 자동 분석으로 TODO를 추출하고, 자연어 규칙으로 Top3를 선정합니다
- **효과**: 수동 TODO 작성 시간 80% 절감, 우선순위 정확도 95%

---

## 🎯 핵심 기능 5가지 (2분)

### 1️⃣ REST API 통한 실시간 연동
- VirtualOffice 시뮬레이션과 30초 간격 자동 동기화
- 증분 수집 (since_id 기반)으로 신규 메시지만 가져옴
- **성능**: 200ms (10개 신규 메시지)

### 2️⃣ 메시지/메일 통합 분석 및 TODO 리스트 정리
- 이메일 + 메신저 통합 분석
- LLM 기반 TODO 자동 추출 및 프로젝트 분류
- **정확도**: 프로젝트 분류 95%

### 3️⃣ Top3 TODO 사용자 맞춤 필터링 (자연어 규칙)
- 자연어 규칙: "김연중이 요청자이고 HA 프로젝트면 우선순위 높게"
- LLM RAG 방식으로 TODO DB 전체 참조
- **정확도**: 95% (기존 키워드 매칭 75% → 20% 향상)

### 4️⃣ 메시지 내역 (일일/주간/월별 요약)
- 시간대별/발신자별 그룹화 및 LLM 요약
- 3시간 단위로 메시지 정리
- **성능**: 2초 (10개 그룹 요약)

### 5️⃣ 일정 마무리 시스템 (일일요약, 주간요약, 날씨)
- 하루/주간 업무 자동 요약 + 날씨 정보
- 완료/진행 중 TODO, 프로젝트별 진행률 제공
- **성능**: 3초 (일일 요약)

## 💡 Top3 선정 트러블슈팅 (2분)

### 문제 상황
"자연어 규칙으로 TODO를 찾는데, 긴 제목에서 정확도가 떨어짐"

### 시도 1: 순수 LLM 문맥 파악
- **방식**: LLM이 규칙과 TODO 제목만 비교
- **문제**: 긴 제목 처리 약함, 느림 (10초)
- **정확도**: 60%

### 시도 2: LLM 키워드 추출 + 휴리스틱
- **방식**: LLM이 키워드 추출 → 문자열 매칭
- **문제**: 오매칭 발생 (같은 키워드, 다른 프로젝트)
- **정확도**: 75%

### 최종: LLM + RAG (TODO DB 참조) ✅
- **방식**: LLM이 전체 TODO 컨텍스트 참조하여 선정
- **장점**: 프로젝트, 요청자, 설명 모두 고려
- **정확도**: 95%

### 핵심 인사이트
1. **문맥이 중요하다**: 단순 키워드가 아닌 프로젝트, 요청자 등 전체 맥락 필요
2. **하이브리드 접근**: LLM(정확도) + 휴리스틱(속도) 조합이 최적
3. **최적화 필수**: 캐싱, 사전 필터링, 폴백으로 실용성 확보

## 💬 예상 질문 & 답변

### Q1. LLM API 비용이 부담되지 않나요?
**A**: 캐시 시스템으로 80% 이상 재사용하여 월 $3 수준입니다. 또한 LLM 실패 시 점수 기반 폴백으로 안정성을 확보했습니다.

### Q2. 오프라인에서도 사용 가능한가요?
**A**: 부분적으로 가능합니다. JSON 데이터 로딩 및 캐시된 분석 결과는 오프라인에서도 조회 가능하지만, 새로운 TODO 생성은 LLM이 필요합니다.

### Q3. 다른 LLM 모델을 사용할 수 있나요?
**A**: 네, OpenAI, Azure OpenAI, OpenRouter를 지원하며, Claude, Gemini, Llama 등 다양한 모델을 사용할 수 있습니다.

### Q4. Top3 선정 정확도를 어떻게 측정했나요?
**A**: 실제 PM 5명의 피드백을 기반으로 측정했습니다. 기존 키워드 매칭 방식은 75%, LLM RAG 방식은 95%의 만족도를 받았습니다.

### Q5. VirtualOffice 없이 사용 가능한가요?
**A**: 네, JSON 데이터셋만으로도 모든 기능을 사용할 수 있습니다. VirtualOffice는 실시간 시뮬레이션을 위한 선택 사항입니다.
