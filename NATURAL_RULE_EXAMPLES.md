# 자연어 규칙 설정 예시

## 🎯 프로젝트 기반 규칙

### Care Connect 프로젝트 우선
```
Care Connect 프로젝트에서 미팅참석 유형을 우선순위 높게
```

### HealthCore API 프로젝트 우선  
```
HealthCore API 프로젝트에서 전형우님 요청 우선
```

### WellLink 브랜드 캠페인 우선
```
WellLink 브랜드 런칭 캠페인 관련 TODO 우선
```

### CareBridge 통합 프로젝트 우선
```
CareBridge Integration 프로젝트에서 테스트 유형 우선
```

## 👤 요청자 기반 규칙

### 특정 요청자 우선
```
전형우님이 요청한 TODO를 우선순위 높게
```

### 이메일 도메인 기반
```
@koreaitcompany.com 도메인에서 온 요청 우선
```

## 📅 마감일 기반 규칙

### 마감일 임박 우선
```
마감일이 임박한 TODO를 우선순위 높게
```

### 오늘 마감 우선
```
오늘 마감인 TODO를 최우선으로
```

## 🏷️ 유형 기반 규칙

### 미팅 참석 우선
```
미팅참석 유형의 TODO를 우선순위 높게
```

### 문서 작업 우선
```
문서작성이나 문서검토 유형을 우선으로
```

## 🔄 복합 조건 규칙

### 프로젝트 + 유형
```
Care Connect 프로젝트에서 미팅참석 유형을 우선순위 높게
```

### 프로젝트 + 요청자
```
HealthCore 프로젝트에서 전형우님 요청을 우선으로
```

### 요청자 + 유형
```
정지원님이 요청한 문서검토 유형을 우선으로
```

### 프로젝트 + 마감일
```
Care Connect 프로젝트에서 마감일이 임박한 TODO 우선
```

## 📊 우선순위 기반 규칙

### 높은 우선순위 우선
```
우선순위가 높은 TODO를 최우선으로
```

### 특정 우선순위 제외
```
낮은 우선순위 TODO는 제외하고 선정
```

## 💡 사용 팁

1. **명확한 표현**: "Care Connect" 보다는 "Care Connect 프로젝트"로 명시
2. **한국어 자연어**: "우선순위 높게", "최우선으로", "먼저" 등 자연스러운 표현 사용
3. **복합 조건**: "그리고", "에서", "중에서" 등으로 조건 연결
4. **프로젝트 이름**: 정확한 프로젝트명 또는 줄임말 사용 (CC, HA, WELL, CI, WI)

## ⚠️ 주의사항

- LLM 연결이 실패하면 자동으로 점수 기반 선정으로 폴백됩니다
- 규칙에 맞는 TODO가 없으면 빈 결과가 반환될 수 있습니다
- 너무 복잡한 조건은 LLM이 이해하지 못할 수 있습니다
